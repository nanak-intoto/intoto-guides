Feature overview

From event idea to attendance proof.

Event Management lets a university run student-facing events in one place. Staff can create events, route them for approval, publish them to students, collect registrations, scan QR codes at entry, and review attendance and feedback after completion.

What the client gets

When Active, the module unlocks the Event dashboard, My Events, Registered Events, Upcoming Events, Pending Events, Create Event, draft handling, approval flow, registration questions, QR attendance, ratings, feedback, and community sharing.

FAQs

Event Management questions clients usually ask.

What is Event Management?
Event Management is the paid module for creating, approving, publishing, registering, attending, and reviewing campus events. It turns an event from an announcement into a full operational workflow.
What information is entered when creating an event?
Creators enter event name, description, start/end date and time, location or virtual link, category, event type, maximum participants, event image, coordinators, and optional registration questions.
Can an event be saved before it is ready?
Yes. Save as Draft keeps the event private and editable until the creator submits it for approval. Drafts do not appear to students as upcoming events.
Who approves events?
University Event Coordinators approve or reject submitted events based on details, timing, capacity, safety, audience fit, and university policy. University Super Admins can also provide oversight where configured.
When does an event appear to students?
Only after it is approved. Approved events appear in Upcoming Events and related dashboards, while rejected or draft events remain hidden from normal discovery.
Can creators add registration questions?
Yes. Creators can add questions for preferences, expectations, consent, dietary notes, eligibility, or planning details. Required questions must be answered before registration completes.
Can users register for events?
Yes. Users open approved event details, review timing/location and any questions, then register. The registration creates the basis for attendance tracking.
Can coordinators verify actual attendance?
Yes. Registered users receive a QR code, and coordinators scan that code to mark attendance. This gives the university proof-based attendance instead of manual estimates.
What happens if a registered user does not attend?
The event can keep registration and attendance as separate states. A user can be registered but not attended if their QR code is not scanned or attendance is not confirmed.
Can events be shared to communities?
Yes, where Community is enabled and sharing is allowed, event details can be shared as event cards into eligible communities. This helps promote events to the right audience.
Can events be physical, virtual, or hybrid?
Yes. Event type supports physical, virtual, and hybrid models where configured. The event record captures either a physical location, a virtual link, or both.
Can attendees provide feedback?
Yes. Completed events can collect ratings and written feedback. Creators and coordinators can review that feedback to improve future planning.
What event statuses are important?
Common statuses include Draft, Pending Approval, Approved, Upcoming, Registered, Assigned, Completed, Rejected, and Attended/Not Attended. These states help staff and users understand where the event sits in the lifecycle.
How does Event Management help staff?
Staff can manage creation, approval, registration, attendance, and feedback from one place. This avoids separate forms, spreadsheets, QR tools, and post-event survey processes.
What happens when inactive?
Event dashboard, create event, draft flow, approval queue, registration, QR attendance, community event sharing, ratings, feedback, and event reporting surfaces are hidden for that university.
