Feature overview

The module every tenant needs before inviting people.

User Management lets a university control who enters Intoto and how many users are allowed under the contract. It supports individual invites, bulk invite/import, role assignment, profile review, suspension, and utilization reporting.

What the client gets

When Active, the module unlocks Manage User, Invite Data, Profile Review, Add User, bulk invites, capacity tracking, domestic/international split where configured, and dashboard people counts. When inactive, user onboarding tools are unavailable.

FAQs

User Management questions clients usually ask.

What is User Management?
User Management is the paid module for inviting users, assigning roles, controlling licensed capacity, reviewing profiles, and managing user access across the university tenant.
Is User Management required to invite users?
Yes. It unlocks Manage User, Invite Data, Add User, bulk invite, profile review, and capacity tracking. Without it, the university cannot run normal onboarding inside Intoto.
What information is used for an invite?
Invite data can include name, email, role, campus, student/staff type, domestic or international bucket, cohort, and other metadata the university needs for onboarding.
Can admins invite users one by one?
Yes. The Add User flow supports inviting individual users when the university needs precise control for a student, staff member, coordinator, or ambassador.
Can admins invite users in bulk?
Yes. Bulk invite or Invite Data supports large intake waves, staff groups, or cohort onboarding. This is important for universities that need to invite hundreds or thousands of users.
What roles can be assigned?
Roles can include Student, Ambassador, External Ambassador, Community Coordinator, Event Coordinator, Third-Party Coordinator, Third-Party Sub Coordinator, Mobility Program Admin, University Admin, and University Super Admin where the tenant has those modules enabled.
How does capacity work?
Capacity is the number of users the university has licensed. The module tracks invited or consumed seats against that limit and shows utilization so admins know how much capacity remains.
What happens when capacity is reached?
The system prevents uncontrolled onboarding. The university can increase capacity commercially or reduce active invited users according to policy.
What are domestic and international buckets?
Some tenants may track domestic and international user limits separately. This helps universities manage different intake groups or commercial tiers while still showing total utilization.
What is Profile Review?
Profile Review is the admin process for checking completed user profiles, required fields, academic or staff details, missing data, and suspicious or incorrect records before the user is fully trusted operationally.
Can admins suspend users?
Yes. Suspension lets the university control access when a user graduates, leaves, changes role, violates policy, or no longer belongs in the tenant.
Does User Management control paid feature access?
User Management controls people and seats. Other paid features control module functionality. A user can only access feature-specific tools when both the role and the related module are available.
Can the university see utilization on the Features card?
Yes. The User Management card can show total capacity, utilization, invited count, and domestic/international usage where configured. This gives commercial visibility to the University Super Admin.
Why is User Management usually purchased first?
Most other modules need people to be invited before they can be used. User Management is the foundation for onboarding students, staff, ambassadors, coordinators, and partner roles.
What happens when inactive?
User invitation, bulk invite, directory management, profile review, capacity tracking, role onboarding, and access lifecycle surfaces are unavailable for the university.
