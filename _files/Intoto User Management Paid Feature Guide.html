Feature overview

The module every tenant needs before inviting people.

User Management lets a university control who enters Intoto and how many users are allowed under the contract. It supports individual invites, bulk invite/import, role assignment, profile review, suspension, and utilization reporting.

What the client gets

When Active, the module unlocks Manage User, Invite Data, Profile Review, Add User, bulk invites, capacity tracking, domestic/international split where configured, and dashboard people counts. When inactive, user onboarding tools are unavailable.

Roles depend on invite and seat control

Sales story: User Management is the commercial control point. It lets the university see exactly how much of the purchased capacity is used and prevents uncontrolled onboarding beyond the contract.

What admins can do

Every invitation and role begins here.

Invite

Add one user

Admins invite a person with email, name, role, campus, student/staff type, and required profile fields.

Bulk

Upload invite data

Bulk invite supports intake waves so universities can onboard large cohorts without manual entry.

Roles

Assign correct access

Users receive role-specific access such as Student, Ambassador, Coordinator, Third-Party Coordinator, or admin roles.

Capacity

Enforce seat limits

The system tracks invited users against purchased capacity and can block or warn when the limit is reached.

Review

Profile review

Admins inspect submitted profiles, verify required details, and resolve incomplete or incorrect records.

Suspend

Control access lifecycle

Admins can suspend or manage users when access should change after intake, graduation, or policy action.

FAQs

User Management questions clients usually ask.

Is User Management required to invite users?
Yes. It unlocks Manage User, Invite Data, Add User, bulk invite, profile review, and capacity tracking.
What happens when capacity is reached?
The system prevents uncontrolled onboarding. The university can increase capacity commercially or reduce active invited users according to policy.
What happens when inactive?
User invitation, directory, profile review, and onboarding surfaces are unavailable for the university.
