# Intoto — Ambassador Platform
## Client Demo Guide

**Design reference:** [Intoto Wireframe — Ambassador (page 5051:661)](https://www.figma.com/design/BSt8IrK987At3ZBXfyUbyY/Intoto-wireframe?node-id=5051-661&p=f&t=Fqpnfj5F0UbdlVZB-0)  
**Prototype:** [Ambassador prototype](https://www.figma.com/proto/BSt8IrK987At3ZBXfyUbyY/Intoto-wireframe?node-id=5051-661&p=f&t=Fqpnfj5F0UbdlVZB-0&scaling=min-zoom&content-scaling=fixed&page-id=5051%3A661)  
**Audience:** Client demos, stakeholders, pre-sales  
**Demo length:** 12–18 minutes  
**Version:** 2.0

---

## 1. Demo objective

Show how the **Ambassador Platform** enables peer mentoring inside Intoto:

- ambassadors build a **public mentor profile** (especially **Ambassador Info**)
- students **discover** mentors on Home and start **Chats** without leaving the app
- ambassadors use a **mentor dashboard** (Quick Way, shortcuts) — not admin governance
- ambassadors **participate** in events and communities as engaged guides
- **Student + Ambassador** dual roles work on one account

## 3. Talk track (30 seconds)

> "The Ambassador Platform is Intoto's human layer for onboarding — experienced students who answer real questions in chat.  
> They maintain a mentor profile students see on day one, while universities keep governance in separate admin roles.  
> The loop is simple: complete profile → appear on student Home → chat → support."

## 4. Step-by-step demo flow

### Step 1 — Login and platform context (1–2 min)

**Do**
1. Log in as **Ambassador**.
2. Point to university **logo + name** (left) and **bell** (right).
3. Show **role tag** — "Ambassador" or "External Ambassador".
4. If dual role: show **role switcher** — "Same person can be Student and Ambassador."

**Say**
- "Scoped to one university — they represent the institution to newcomers."
- "Not a super-admin panel — no invite staff or approve communities here."

---

### Step 2 — Ambassador Home dashboard (2 min)

**Do**
1. Scroll Home — Quick Way, Quick Actions, banners, Upcoming Events.
2. Highlight **Total Messages** and **Total Students** (if shown).
3. Tap university header → **University details** → back.
4. Pull to refresh once.

**Say**
- "Dashboard is API-driven — mentor stats and shortcuts, not governance counts."

---

### Step 3 — Profile completion and Ambassador Info (3–4 min)

**Do**
1. Tap **profile completion alert** (or Profile → My Profile).
2. Show section list — point to **Ambassador Info** as mandatory public section.
3. Open **Ambassador Info** — fill bio, languages, programs (2–3 fields).
4. **Save** → hub shows completed state.
5. Optional: update **profile photo** from header.
6. Return Home — alert reduced or gone after refresh.

**Say**
- "Students only trust complete cards — this section drives the public mentor profile."
- "Incomplete profiles are deprioritized on the student Ambassadors carousel."

---

### Step 4 — Chats (2–3 min)

**Do**
1. Open **Chats** tab (or **Total Messages** Quick Way).
2. Open a thread; send a helpful reply.
3. Mention **Support** for visa, conduct, or official policy questions.

**Say**
- "Primary job: fast, in-app answers before arrival."

---

### Step 5 — Student discovery loop (2–3 min) — strongly recommended

**Do**
1. On **second device**, log in as **Student**.
2. Home → **Ambassadors** carousel → tap demo ambassador.
3. Show **public profile** → tap **Chat**.
4. On ambassador device, show message arrives (notification + Chats).

**Say**
- "Closed loop — student finds mentor on Home, never leaves Intoto."
- "View All on Ambassadors section opens full ambassador directory."

---

### Step 6 — Students list (1 min) — if configured

**Do**
1. **Total Students** Quick Way → search one name → open profile → **Chat**.

**Say**
- "Browse scope for proactive outreach — still not admin user management."

---

### Step 7 — Events and Community (1–2 min)

**Do**
1. Quick Action **Events** → open one event → **Register** (optional).
2. **Community** tab → open one community feed briefly.

**Say**
- "Ambassadors model engagement — they don't manage the event approval queue."

---

### Step 8 — Profile tab and close (1 min)

**Do**
1. Profile tab → Settings / Language / FAQs (brief).
2. Recap: profile + chat + student visibility.

**Say**
- "Ambassador Platform scales peer support without adding admin headcount."

## 7. FAQ for clients

| Question | Answer |
|----------|--------|
| Ambassador vs Student? | Student = own journey; Ambassador = help others + public mentor profile. |
| External Ambassador? | Same platform; invited with ambassador type; may not be enrolled student. |
| Can ambassadors invite staff? | No — University Super Admin / coordinators handle invites. |
| Customize dashboard? | Yes — backend overview config per role. |
| How do students find ambassadors? | Home **Ambassadors** carousel and **View All** list. |

---

## 8. Do not demo on Ambassador role

- Manage Users, Add User, Assign Students
- Approve pending events or communities
- University Super Admin Quick Way (all campuses, all invites, reports)

Switch role explicitly if the demo account is multi-role.
