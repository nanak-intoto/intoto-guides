FAQs

Mobility Program questions clients usually ask.

What is Mobility Program?
Mobility Program is the paid module for managing academic exchange operations. It covers applications, document review, approval decisions, nominations, programs, partners, mobility reports, and university-level oversight.
Is Mobility Program separate from normal User Management?
Yes. User Management controls invited users and seats. Mobility Program adds the specialized exchange workflow: applications, approvals, nominations, programs, partners, documents, and reports.
What information is collected from a student application?
The application can include student profile, program choice, destination preference, GPA, major, academic year, statement of purpose, transcripts, passport or identity files, and required mobility documents.
Who approves applications?
Mobility Program Admins review applications and decide approve, reject, or request more information. They focus on day-to-day eligibility and operational decisions.
How does an admin review an application?
The admin opens the application detail page and checks completeness, GPA, documents, eligibility, destination/program fit, and review notes. High-priority applications can be highlighted with badges where configured.
What happens when an application is approved?
Approved applications become eligible for partner nomination. The student's journey moves forward and the admin can assign the student to a partner university from the Nomination Hub.
What happens when an application is rejected?
The application stops at the rejected path, and nomination is not available. The student sees the status according to the configured student-facing experience.
What does request more information mean?
It means the admin cannot make a final decision yet because something is missing or unclear. The student is asked to provide the missing document, clarification, or correction.
What does nomination mean?
After approval, the admin selects a partner university and nominates the student. The student then waits for the partner university acceptance stage.
What does the Nomination Hub show?
The Nomination Hub shows approved students who are eligible for nomination, nominated vs pending counts, nominate actions, and partner university selection. It helps admins avoid nominating students before approval.
What program information can be managed?
Programs can include type, duration, deadline, funding, partner universities, enrollment, status, and a detailed description. This gives admins a practical portfolio view.
What partner information can be managed?
Partner university records can include country, city, agreement status, student counts, ranking/context, active programs, website, and operational notes. This context supports nomination and reporting.
What reports are available?
The module can include annual, quarterly, evaluation, survey, and financial reports. Reports summarize mobility activity, completion, satisfaction, budget, and partner performance depending on the configured data.
How is University Admin different from Mobility Program Admin?
Mobility Program Admins run daily operations: applications, approvals, nominations, programs, partners, documents, and reports. University Admins monitor institutional strategy, budget, network health, programs, users, and leadership-level reporting.
What happens when the module is inactive?
The Mobility Program Admin role, command center, approvals, nominations, programs, partners, mobility documents, mobility reports, and institutional mobility oversight surfaces are hidden.
