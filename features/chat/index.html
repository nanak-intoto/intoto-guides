Feature overview

Support conversations without personal phone numbers.

Chat gives universities controlled messaging across their Intoto community. Students can contact the right person, staff can create groups, and conversations stay attached to university roles instead of scattered across personal apps.

What the client gets

When Active, Chat unlocks mobile Chats tab, web Chat sidebar, direct messaging, group creation, inboxes, message threads, read/unread states, and contextual sharing where other modules allow it. When inactive, chat surfaces are hidden.

FAQs

Chat questions clients usually ask.

What can users send in Chat?
Users can send text messages and, where enabled, audio, video, documents, single images, multiple images, shared events, shared community posts, shared communities, and profile or user context. Actual availability still depends on the app configuration, role permissions, and the other modules purchased by the university.
Can users send documents in Chat?
Yes. Chat can support document attachments such as PDFs or other supported file types for letters, forms, proof files, or instructions. This is useful for support follow-up because the file stays with the conversation history.
Can users send one image or multiple images?
Yes. A user can share a single image, such as a screenshot or campus photo, and can also share multiple images where the app supports gallery-style attachments. This helps students show a problem clearly instead of describing it only in text.
Can Chat support audio messages?
Chat can support audio or voice-note style messages where enabled. This is helpful for students who are more comfortable speaking, for quick explanations, or for language-support scenarios.
Can Chat support video messages?
Chat can support video attachments where enabled by the app configuration. Video is useful for richer explanations, orientation content, or showing a problem that is hard to explain with text or images.
Can events be shared in Chat?
Yes, when Event Management and Chat are Active and sharing is allowed. An event can be shared as a contextual card so recipients see the event information without copying the event manually.
Can Community posts be shared in Chat?
Yes, where Community and Chat are Active and sharing is allowed. A community post can be sent into a direct or group conversation so a student, coordinator, or support team can discuss it with context.
Can a community itself be shared in Chat?
Where the app exposes community sharing, users can send a community reference into chat. This helps staff guide a student to the right group without asking them to search manually.
Is Chat required for Community or Ambassador conversations?
Community and Ambassador Management can expose people, posts, groups, and profiles, but in-product messaging requires Chat to be Active. Without Chat, users may still view the relevant surfaces, but the message action is hidden or unavailable.
Can staff create group conversations?
Yes. Web chat supports group creation for relevant users and operational groups where permitted. Groups can be used for cohorts, events, ambassador teams, support teams, or university staff coordination.
Can students message ambassadors?
Yes, when both Chat and the relevant Ambassador surfaces are Active and the role permissions allow it. A student can move from an ambassador profile to a conversation instead of using personal contact details.
Does Chat replace email?
Chat does not need to replace every formal email. It is best for fast support, peer guidance, operational follow-up, and contextual discussion around features such as events, communities, and mobility support.
How does Chat help university staff?
Staff can keep student conversations in one university-owned product, create group conversations, share module context, and avoid fragmented communication across private channels. Conversation history also makes follow-up easier for teams.
What controls who can message whom?
Messaging depends on the university's Chat entitlement, role permissions, privacy rules, and the entry point used to start the conversation. The product should only expose compose actions where the sender and recipient relationship is allowed.
What happens when Chat is inactive?
Chat sidebar, mobile Chats tab, compose paths, inboxes, group creation, direct messages, content sharing into conversations, and in-product messaging are hidden for that university.
